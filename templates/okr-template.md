# OKR Template

**Framework:** Objectives and Key Results  
**Cadence:** Quarterly · **Review:** Monthly check-in

---

## What Makes a Good OKR

**Objective:** Qualitative, aspirational, time-bound. Answers "where are we going?"  
**Key Result:** Quantitative, outcome-based (not output). Answers "how will we know we got there?"

**The test for a Key Result:** Can you score it 0.0–1.0 with zero ambiguity? If yes, it's a KR. If it needs judgment to grade, it's not specific enough.

---

## Grading Guide

| Score | Meaning |
|---|---|
| 1.0 | Fully achieved — consider whether target was too conservative |
| 0.7 | Sweet spot — ambitious but realistic |
| 0.5 | Partial progress — acceptable for stretch goals |
| 0.3 | Significant miss — investigate root cause |
| 0.0 | No progress — something fundamentally changed or was wrong |

**Target grade is 0.7, not 1.0.** Consistently hitting 1.0 means targets are sandbagged. Consistently hitting 0.3 means targets are fantasy.

## Common Mistakes

| Mistake | Why It Fails | Fix |
|---|---|---|
| KR = Task ("Launch feature X") | Tasks are binary — done or not. No signal on impact. | Replace with outcome: "Increase checkout completion from 60% to 75%" |
| Sandbagged targets | Team hits 1.0 every quarter → no learning, no ambition | Set targets with ~30% expected failure |
| Too many OKRs | Everything is a priority = nothing is | Max: 1 objective, 3 KRs per team per quarter |
| OKRs disconnected from strategy | Team optimizes locally, misses company goal | Every KR should trace to a company-level outcome |
| Grading on effort, not outcome | "We worked hard, so 0.8" | Grade the number, not the effort |
